# Terminology Editing Round: Preference Formation Over Closure

Date: 2026-06-25

## Objective

Demote "closure" from reader-facing prose because it is not a settled game-theory
term and was making the title, subtitle, abstract, tables, and section prose
sound more specialized than the idea requires.

## Editorial Decision

- Use "preference formation," "preference-forming rule," "settling," and
  "settled preference state" in the main text.
- Keep "closure law" and "closure map" only in the formal appendix, after an
  explicit definition: in this project, closure means the limiting rule that
  records where fast preference adjustment settles.
- Avoid "closure" in the title, subtitle, abstract, section headings, table
  headings, and ordinary explanatory prose.

## Changes Made

- Changed the subtitle to "Fast preference formation..."
- Removed "closure law" from the abstract.
- Replaced main-text uses such as "after closure," "closure loop,"
  "fast-closure mechanism," and "closure risk" with ordinary-language
  equivalents.
- Kept the formal map \(C_\ell\) in Appendix A and defined the terminology
  there.
- Updated the standing agent objective function so later editing rounds preserve
  this distinction.

## Check

The remaining uses of "closure" should be confined to the formal appendix and
should be understood as project-specific technical shorthand, not standard game
theory terminology.
