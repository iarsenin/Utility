# Round 11 Modeller Memo

## Verdict

The current drafts are technically much stronger than the earlier versions, but the exposition still makes the reader work too hard to learn the result. The abstract and introduction begin with correct operator language, then only later reveal the substantive claim. This is backwards for a broader economics audience. The paper should say, near the top: if preferences adapt infinitely fast, Nash equilibrium is still well defined, but it is computed in the game produced by the preference-formation process; material selection cannot distinguish initial preference states that collapse to the same post-closure utility; selection therefore moves to the laws, platforms, institutions, or environments that generate those utilities.

This can be made friendlier without weakening rigor. The main change is ordering: conclusion first, conditions second, caveats third.

## Results To State Upfront

1. Fast preferences do not destroy Nash equilibrium. They change the payoff table to which Nash equilibrium is applied.

2. Utility stops being a primitive. In the fast limit, utility is an output of the closure law \( \ell \) and slow environment \(E\): the object agents maximize is \(U(\cdot;\theta_\ell^\star(E),E,\ell)\).

3. Nash predictions are robust only in a special case: closure must preserve mixed best-response correspondences. Affine material transformations with opponent-only terms satisfy this; generic strategic distortions do not.

4. Selection over initial preference states can disappear in the limit. If many initial states flow to the same post-closure state, any selection rule that observes only post-closure play assigns them the same material value.

5. Darwinian/evolutionary logic therefore shifts target. It acts on whatever remains heterogeneous after closure: closure laws, institutions, platform rules, environments, or populations carrying different preference-producing mechanisms.

6. Harm is not automatic. It requires misalignment between the rule that produces/selects preferences and the material or welfare evaluator.

## Rigor Boundaries

Keep the fixed-\(E\), fixed-\(\ell\) boundary explicit, but do not lead every paragraph with it. A reader can absorb this as a maintained scope condition after the main conclusion is stated.

Do not imply Proposition 2 proves Nash-set separation in all games. It proves best-response non-invariance; Example 1 shows actual Nash-set change in a minimal two-action construction.

Do not say selection disappears. Selection over collapsed initial states is degenerate under post-closure admissible rules. Selection over laws remains active once a law-level evaluator and dynamic are specified.

Do not say platform optimization is inherently harmful. Proposition 3 is an alignment result: exact proxy/material maximizer alignment gives no material loss on the feasible set; two-rule reversal gives loss on that feasible set.

Do not oversell WDI in the working paper. It is a measurement prototype, not causal evidence that social media or AI changes preferences.

## Places Where Jargon Hides The Point

The GEB abstract opens with "finite games in which the utility representation used for play is generated by a fast preference state." This is precise but not gripping. Start instead with the extreme-limit result in ordinary language, then formalize.

The GEB introduction's "operator order" paragraph is accurate but too abstract. It should be preceded by a plain-language paragraph: "First preferences settle, then players optimize, then the analyst evaluates material consequences."

Section 4 says the lemma is not a welfare result, but it should also say what it does positively: it licenses replacing the original preference state by the post-closure game.

Section 5 should open with the punchline: Nash survives as a method; the Nash set survives only when closure leaves best responses unchanged.

Section 6 should avoid "material selection is an evaluation operator" as the first sentence. Start with the economic implication: initial preference differences stop mattering if fast closure sends them to the same utility representation.

Section 8 reports useful numbers, but the interpretation paragraph should say the conclusion before the diagnostics: random strategic closure usually changes the game; neutral closure does not; noisy proxy alignment produces smaller losses than independent or reversed proxies.

Section 9 is good, but the first paragraph can be sharper: when a system produces the preferences used to evaluate it, final-preference welfare is not a stable welfare criterion by itself.

The working paper's "What Is Invariant?" table is excellent and should be used as a style model for the GEB draft.

## Writer Instructions

Revise the GEB abstract into a three-move structure: result, conditions, application. Avoid opening with "for each fixed..." or "boundary layer." Put scope conditions in the second half.

In both drafts, add short "Plain implication" or "What the result says" sentences after Lemma 1, Proposition 1, Proposition 2, Theorem 1, and Proposition 3. These should not replace theorem statements; they should tell the reader how to read them.

Move several caveats later in paragraphs. State the result first, then add "The statement is conditional: ..." or "The exception is ..."

Use the invariant/non-invariant table logic earlier in the GEB introduction. The reader should know by page one what survives and what changes.

Replace abstract phrases like "operator order," "post-closure object," and "robust invariance discipline" where they are doing expository work. Keep them where they are formal labels.

Preserve all theorem statements unless changing exposition around them. The mathematics is close; the problem is not correctness but reader access.
